## 🎓សេចក្តីផ្តើមក្នុងការរៀន NextJS

សូមស្វាគមន៍មកាន់ការបង្រៀន NextJS ជាភាសាខ្មែរដែលដកស្រង់ពីឯកសារច្បាប់ដើមរបស់គេហទំព័រផ្លូវការរបស់ <a href="https://nextjs.org/docs" target="_blank">NextJS</a>

#### តើអ្វីទៅដែលគេគ្រប់គ្នាហៅថា NextJS ?

NextJS គឺជា React framework សម្រាប់ធ្វើការអភិវឌ្ឍន និងបង្កើតរូបរាងនៃគេហទំព័រ ឬ កម្មវិធី-គេហទំព័រ (Web Application)។ ដោយយើងប្រើប្រាស់នូវ React Component ដើម្បីបង្កើតរូបរាងនៃ User Interface
និង NextJS សម្រាប់បន្ថែមនូវមុខងារជាច្រើន និងការបង្កើនប្រសិទ្ធភាព (Optimizations)។

NextJS វាមាន Automatic Configures Tools ជាច្រើនដែល React ត្រូវការដូចជា៖ bundling, compliling, និង configures ដ៏ទៃទៀតជាច្រើន។ វាអនុញ្ញតិឲ្យយើងផ្តោតទៅលើការបង្កើត Application ជាជាងឲ្យ
ឲ្យយើងធ្វើការចំនាញពេលក្នុងការ Config ដោយខ្លួនឯង។

ទោះបីជាអ្នកកំពុងអភិវឌ្ឍនគេហទំព័រ-កម្មវិធីដោយខ្លួនឯង ឬអភិវឌ្ឍនជាក្រុមក៏ដោយ, NextJS គឺជួយឲ្យអ្នកធ្វើការបង្កើត Interactive, Dynamic, និង Fast React Application បានយ៉ាងងាយស្រួល និងឆាបរហ័ស។

#### តើមុខងារចំបងរបស់ NextJSមានអ្វីខ្លះ?

មុខងារសំខាន់ៗរបស់ NextJS គឺមានដូចខាងក្រោម៖

| មុខងារ        | បរិយាយមុខងារ                                                                                                                                                                                                                |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Routing       | Routing គឺជាមូលដ្ឋាននៃ File-System ដើម្បីបង្កើតនៅ​ខាងលើផ្នែក Server Component ដែល support layouts, nested routing, loading states, error handling និង features មួយចំនួនទៀត។                                                 |
| Rendering     | Client-side និង Server-side Rendering ជាមួយនឹង Client និង Server Component និងបន្ថែមជាមួយការធ្វើឲ្យប្រសើរឡើង (Optimize) ជាមួយនឹង Static និង Dynamic rendering នៅលើ server របស់ NextJS។​ ការ Streaming និង Node.js runtimes។ |
| Data Fetching | Data fetching ជាមួយនិង async/await នៅក្នុង Server component និង ការ extended fetch API សម្រាប់ការ request memoization, data caching និង revalidation។                                                                       |
| Styling       | គាំទ្រនូវការប្រើប្រាស់វិធីសាស្រ្តក្នុងការសរសេរ styling ដូចជា CSS modules, Tailwind CSS, និង CSS-in-JavaScript                                                                                                               |
| Optimizations | Image, Fonts, និង Script Optimizations សម្រាប់បង្កើននូវភាពឲ្យល្អប្រសើរនៃ Application's core web vitals និង បទពិសោធន៍របស់អ្នកប្រើប្រាស់ (User Experience UX)                                                                 |
| TypeScript    | គាំទ្រក្នុងការប្រើប្រាស់ភាសា TypeScript ដើម្បីជាប្រយោជន៍ក្នុងការឆែកមើល type និងការ compliling ដោយមានប្រសិទ្ធភាព ក៏ដូចជាការ custom typescript plugin និងការឆែក type។                                                         |

#### App Router vs Pages Router

Nextjs មាន router ពីរខុសៗគ្នាគឺ៖ The app router និង pages router។ App router គឺជា router ថ្មីអនុញ្ញាតិឲ្យយើងអាចប្រើប្រាស់នូវមុខងាររបស់ React ដែល latest ដូចជា Server component និង Streaming។
Pages router គឺជា Nextjs មុងងារ router ដើម ដែលអនុញ្ញតិឲ្យយើងធ្វើការបង្កើត Server-Rendered React Applications និង បន្តធ្វើការ Support សម្រាប់ NextJS application សេរីចាស់។

ក្នុងការបង្រៀនក្នុងថ្នាលសិក្សាមួយនេះគឺពួកយើងផ្តោតតែទៅលើការប្រើប្រាស់មុខងារ App Router ប៉ុន្នេាះជាដំណាក់កាលបន្ទាប់ពួកយើងនឹងព្យាយាមក្នុងការចងក្រងអត្ថបទជាភាសាខ្មែរមួយទៀតសម្រាប់ Pages Router នៅពេលអនាគត។

ដូច្នេាះសូមចាប់ផ្តើមរៀនជាមួយពួកយើងក្នុងការបង្កើតគេហទំព័រ-កម្មវិធីដោយប្រើប្រាស់នូវ App Router របស់ NextJS និងយល់ដឹងអំពីមុខងារមួយចំនួនដែលមាននៅក្នុង NextJS 14 សូមរៀនដោយយកចិត្តទុកដាក់សូមអគុណ
